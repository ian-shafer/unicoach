# Skill: RFC Implementation Fix

This skill takes the output of an `rfc-impl-review` report and acts as the strict "fixer." It implements the explicitly recommended options from the review's "Action Items for Implementor" section.

## Critical Behaviors

- **Context Inheritance**: You execute within the same context as the original `rfc-impl` skill. You must ensure you have ingested the target RFC, relevant `SPEC.md` files, and required defensive coding skills before modifying code.
- **Strict Adherence**: You MUST strictly implement only the fixes recommended in the report. Do NOT introduce speculative changes, extra refactoring, or feature creep outside the bounds of the review feedback.
- **NO GIT COMMITS**: You MUST NOT make any git commits. Leave all modifications in the local working directory. The architect or external review skills will evaluate the raw uncommitted diffs.
- **Verification**: After applying the fixes, you MUST run relevant tests (e.g., `bin/test` or specific test suites) to ensure your changes compile and pass.

## Execution Flow

1. **Read the Report**: Identify and ingest the "Master Implementation Review" report generated by `rfc-impl-review`. If the report is not explicitly provided in the context, ask the user to provide it.
2. **Apply Action Items**: Parse the "Action Items for Implementor" section. For each item:
   - Review the provided options.
   - You MUST select and implement the explicitly recommended option.
   - If a recommendation requires clarity, use your codebase search tools to verify context before modifying files.
3. **Execute Tests**: Verify the implementation code compiles and all related test suites pass. If tests fail, systematically debug and fix them based only on the boundaries of the action items.
4. **Demonstrate Success**: Upon completing the fixes and verifying a green test suite, explicitly inform the architect that the fixes are complete and ready for their review or subsequent automated verification cycles.
